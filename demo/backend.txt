@Controller : 해당 클래스가 컨트롤러 역할임을 명시
    - 프레임워크가 해당 클래스를 관리함 (new 한 후 관리)
    - 경로 중복 금지

템플릿 : html, java 코드가 사용 가능해야 함
 => thymeleaf에서 java코드 사용법 

  <p> [[${info.addr}]]</p>
  <p> [(${info.addr})]</p>
  <input th:value="${info.addr}"/>
  <p th:text="${info.addr}"></p>


  주요 어노테이션
  @Controller
  @RequestParam : http 요청의 파라미터를 컨트롤러 메서드의 매개변수로 바인딩(연결)
   public void postCalc(int num1, int num2) {} : 이름을 똑같이 한다면 @RequestParam 생략 가능
   public void postCalc(@RequestParam int num1, @RequestParam int num2) {}
   
   이름이 다를 경우
   public void postCalc(@RequestParam("num1") int number1, @RequestParam("num2") int number2) {}

   파라메터로 값이 안들어오는 경우 400 에러 발생
   public void postCalc(@RequestParam(required = false, defaultValue= "0") int num1, @RequestParam(required = false, defaultValue="0") int num2) {}

   @RequestMapping() : 예전에 get / post 시 사용했지만 지금은 default 경로를 지정해주는 용도로 사용됨


Model : 컨트롤러가 뷰에 데이터를 전달하기 위해 상요하는 인터페이스

model.addAttribute("키",값) 

@ModelAttribute("키") == Model : 뷰로 데이터 전달 + (데이터 바인딩(연결))


RedirectAttributes : 리다이렉트(return "redirect:/home")가 발생한 후 데이터를 일회성으로 전달하기 위해 사용하는 인터페이스
    1. 주소줄 이용 : rttr.addAttribute("bno",10);
    2. HTTP session을 임시로 사용 : rttr.addFlashAttribute("money",1000);
